# 06. 기여도 산정 로직

기여도는 **4종류 점수**로 구성된다. 회의에서의 활동(발언·참석)과 맡은 일(액션)을 분리해 측정하고, 최종에서 합산한다.

## 설계 원칙

1. **4종류 점수 분리** — 회의 기여도·태스크 기여도·회의 종합 기여도·종합 기여도를 독립 산출
2. **비율 기반** — 회의 길이·팀 규모와 무관하게 비교 가능
3. **기회 보정** — 측정 불가 축은 0점 처리 금지, 가중치 재분배
4. **명시적 보정** — 모든 보정에 라벨 표시
5. **팀별 설정** — 최종 합산 비율 등 주요 값은 TeamSettings에서 조정 가능

---

## 점수 4종류

| 번호 | 종류 | 설명 | 입력 |
| --- | --- | --- | --- |
| 1 | 회의 기여도 | 단일 회의의 발언 + 참석 | 단일 회의 데이터 |
| 2 | 태스크 기여도 | 전체 기간 액션 아이템 기반 | 전체 기간 액션 목록 |
| 3 | 회의 종합 기여도 | 여러 회의를 시간 가중 평균으로 누적 | 회의 기여도 목록 |
| 4 | 종합 기여도 | 회의 종합 + 태스크를 합산한 최종 점수 | 3번 + 2번 |

핵심: 회의에서의 활동(발언·참석)과 맡은 일(액션)은 **완전히 분리**해 계산하고, 4번에서만 합산한다.

---

## 1. 회의 기여도

단일 회의에서 한 팀원의 발언 비중과 참석 성실도를 측정한다.

### 발언 비중 축 (기본 75%)

```
유효 글자수  = min(본인 발언 글자수, action_chars_limit × 발언 횟수)
실제 비중    = 유효 글자수 / (본인 참여 시간 내 전체 참여자 글자수)
균등 기대치  = 1 / 팀원 수 (N)
발언 점수    = min(1.0, 실제 비중 / 균등 기대치)
```

- 팀원 수 기준 균등 기대치(1/N) 이상 발언하면 만점
- 발언 1회당 글자수 상한(`action_chars_limit`)을 적용해 독점 방지
- 본인이 있던 시간대의 글자수만 분모로 사용 (참여 시간 보정)
- 참석만 하고 말 안 한 사람도 인원수(N)에 포함 → 본인 0점, 남들 기대치 소폭 하락

### 참석 축 (기본 25%)

```
출석 비율  = 실제 참여 시간 / 회의 총 시간
허용 지각  = 회의 길이 × punctuality_grace_ratio (기본 10%)
정시 점수  = max(0, 1 - 지각 시간 / 허용 지각)
참석 점수  = (출석 비율 + 정시 점수) / 2
```

- 지각이 허용 한도를 초과할수록 정시 점수가 선형 감소
- 허용 한도 2배 이상 지각 시 정시 점수 0
- 출석 비율과 정시 점수를 평균내므로, 지각이 심해도 참석 점수가 0까지 내려가지 않음

### 가중치 재분배

```
회의 기여도 = Σ(축 점수 × 가중치) / Σ(적용된 가중치)
```

측정 불가 축(발언 데이터 없음 등)은 분모에서 제외 → 남은 가중치가 비례 확장.

### 저참여 플래그

참여율이 `min_attend_ratio`(기본 40%) 미만이면 `low_attend_flag=True` 로 표시한다.
점수 추가 하향은 하지 않는다 — 참석 점수 자체에 출석 비율이 이미 반영되어 있기 때문이다.

### 완전 결석

완전 결석(한 번도 참여 안 함) 시 회의 기여도 0점을 즉시 반환하고 신뢰도를 `Absent`로 표시한다.

## 2. 태스크 기여도

전체 기간 동안 배정된 액션 아이템 목록으로 한 번에 계산한다. 회의별로 묶지 않는다.
액션은 각 회의(`MemberMeetingData.actions`)에 저장되고, 태스크 기여도 계산 시 `collect_actions`로 모아서 사용한다.

### 마감 준수 점수표

| 모드 | 마감 전 | 당일(d=0) | 1일 이내 | 3일 이내 | 초과 |
| --- | --- | --- | --- | --- | --- |
| strict  | 100 | 100 | 40 | 20 | 10 |
| normal  | 100 | 100 | 70 | 40 | 20 |
| lenient | 100 | 100 | 100 | 90 | 30 |

- 당일(d=0)은 모든 모드에서 100점 — 기한을 지킨 것이므로 감점 없음
- 미완료 액션은 모드와 관계없이 0점
- 마감 정보 없는 완료 액션은 100점 처리

### 난이도 가중 평균

```
액션 난이도: 1=하 / 2=중(기본) / 3=상
total_weight  = Σ(난이도)
완료율        = Σ(완료 액션 난이도) / total_weight
마감 준수율   = Σ(마감 점수 × 난이도) / total_weight / 100
태스크 기여도 = 완료율 × 0.5 + 마감 준수율 × 0.5
```

액션이 없으면 측정 불가 → 4번 종합에서 가중치 재분배.
`total_weight=0`이면 단순 평균으로 폴백(ZeroDivisionError 방지).

## 3. 회의 종합 기여도 (누적)

여러 회의의 기여도를 회의 시간 가중 평균으로 합산한다.

```
회의 종합 기여도 = Σ(회의 기여도 × 회의 시간) / Σ(회의 시간)
```

### 누적 포함 기준

아래에 해당하면 누적에서 제외 또는 0점 처리한다.

| 조건 | 처리 |
| --- | --- |
| 비정규 회의 (`is_official=False`) | 제외 |
| 최소 회의 시간 미만 (기본 5분) | 제외 |
| 무단 결석 | 0점으로 포함 |
| 사유 결석 | 누적에서 제외 (팀원 전체 동의 필요) |

### 사유 결석 처리

사유 결석은 팀원 전체 동의 하에 승인되며, 승인된 경우 해당 회의를 누적에서 제외한다.
불가항력 보호가 목적이므로 완전 면제 처리한다.

---

## 4. 종합 기여도 (최종)

회의 종합 기여도와 태스크 기여도를 `weight_task_in_final` 비율로 합산한다.

```
종합 기여도 = Σ(점수 × 가중치) / Σ(적용된 가중치)
```

- 태스크 기여도가 None(액션 없음)이면 회의 종합만으로 계산
- `weight_task_in_final` 은 그룹 생성 시에만 설정 가능

### 팀장 보정

```
종합 기여도 × (1 + leader_bonus)   # 기본 0.2
최대 1.0 상한 적용
```

- 팀장의 진행·정리 부담을 반영한 가산
- 4번 종합 기여도에만 한 번 적용

---

## 신뢰도 라벨

| 등급 | 조건 |
| --- | --- |
| High   | 2축 모두 측정 + 회의 30분 이상 + 오디오 손실 5% 미만 |
| Medium | 1축 측정 또는 회의 15~30분 또는 일부 캡처 손실 |
| Low    | 0축 측정 또는 회의 15분 미만 또는 캡처 손실 20% 이상 |
| Absent | 완전 결석 — 측정 데이터 없음 |

LOW 조건을 먼저 확인해 HIGH·MEDIUM보다 우선 적용한다.
Absent는 완전 결석 시 즉시 반환되며 다른 조건과 무관하다.

---

## 처리 정책 우선순위 (점수 산정 시 순서)

1. 데이터 무결성 검증
2. 신원·권한 검증
3. 시간 동기화
4. 사유·예외 처리 (완전 결석 즉시 반환 → Absent 라벨)
5. 측정 가능성 판단
6. 상한 적용 (글자수 상한)
7. 재분배 계산
8. 신뢰도 라벨링

## 팀 설정 가능 항목 (TeamSettings)

| 설정 | 기본값 | 설명 |
| --- | --- | --- |

| `weight_attend_in_meeting` | 0.25 | 회의 내 참석 비중 |
| `weight_task_in_final` | 0.50 | 최종에서 태스크 비중 — **그룹 생성 시에만 설정** |

| `punctuality_grace_ratio` | 0.10 | 허용 지각 비율 (회의 길이 대비) |

| `deadline_mode` | `normal` | 마감 패널티 모드 (strict / normal / lenient) |
| `min_meeting_sec` | 300초 | 누적 포함 최소 회의 시간 |
| `score_visibility` | `all` | 기여도 공개 범위 (all / self / leader) — 표시 정책 |
